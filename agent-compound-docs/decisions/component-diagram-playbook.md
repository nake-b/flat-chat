# Component / Context Diagram Playbook

How to build and iterate the hand-laid `.drawio` diagrams in `services/backend/`
(`component-diagram.drawio`, `context-engineering-diagram.drawio`). These are
crafted by hand, NOT generated by the architecture-diagram skill — the skill's
auto-layout looks generic and the user has rejected it. Edit the XML directly.

## 0. House style — match the existing files, don't reinvent

- **Light theme.** White/`#F8FAFC` cards, dark text (`#0f172a` titles, `#475569`/`#64748b`
  body). The diagram is viewed in draw.io's default LIGHT canvas. **Do NOT set light
  title colours "for dark mode"** — `architecture.png` only looks dark because it's a
  dark *export*; the file itself is light. A light-coloured title is invisible in the editor.
- Lanes/zones = light-tinted rounded rectangles with a title. Cards = white rounded
  rectangles with a brand-coloured stroke.
- **Big fonts** (this gets projected): card titles 16–18px, body 13–14px, edge labels ~13px,
  diagram title 22–26px.
- **Big gaps.** Generous empty corridors between lanes (~80px) are what make arrows readable.
  Don't fill empty space with decoration.
- Keep the colour conventions and mirror them in a legend cell: gray `#64748b` in-process/HTTP/SQL,
  orange `#F59E0B` SSE/state mutation, dashed gray `#94A3B8` context/cross-cutting,
  dashed green `#10B981` optional embed.

## 1. The file is mxGraph XML — edit it like code

```
<mxfile><diagram><mxGraphModel pageWidth pageHeight>
  <root>
    <mxCell id="0"/>
    <mxCell id="1" parent="0"/>
    <!-- everything else parent="1", ABSOLUTE coords -->
  </root>
</mxGraphModel></diagram></mxfile>
```

Boxes = `vertex="1"` with `<mxGeometry x y width height>`. Arrows = `edge="1"` with
`source`/`target` cell ids. Keep `parent="1"` for everything and use absolute coords —
containment becomes purely positional, which makes the overlap-checker trivial.

## 2. I can't see pixels — so RENDER AND LOOK before showing the user

Two complementary checks, both run before every reveal:

1. **PNG render + Read it.** `cairosvg` is available (`pip install cairosvg --break-system-packages`).
   A small script parses the `.drawio` and rasterises boxes + edges (use the edge waypoints)
   to PNG; then Read the PNG to actually see overlaps/legibility. Zoom by rendering at 2× and
   cropping with Pillow. Render in LIGHT (default) since that's how the user views it.
2. **`mcp__drawio__open_drawio_xml`** opens it in the user's browser for the real view.
   (For files with big embedded icons, strip image data first — see
   `editing-drawio-programmatically.md`.)

## 3. Reserve arrow corridors BEFORE placing boxes

- Decide which strips of canvas are "arrow highways" first. e.g. 80px inter-lane gaps for
  cross-lane arrows, and a vertical corridor left of `chat_session` (x≈500) for the long
  `ResultSet → Agent` context arrow.
- **Don't let components fight.** Put `agent_inner` on the LEFT of `agent_unit` and tools on
  the RIGHT so the context arrow enters `agent_inner`'s left side without crossing a tool box.
  Put `msg_history` on the RIGHT of `chat_session` to clear the left for that same arrow.
- **Align components that connect.** Position endpoints so the ones that connect to ChatService
  sit inside its vertical y-range → direct horizontal arrows, no bends.
  `entryY = (source_mid_y − target_y) / target_height` places the entry exactly opposite the source.

## 4. Arrow routing

- `exitX/exitY`, `entryX/entryY` are fractions [0,1] of the box. First segment direction follows
  the edge you exit (`exitX=1`→right, `exitY=1`→down, etc.).
- Add `<Array as="points">` waypoints to force corners when the router would otherwise cross a box.
- `exitPerimeter=0`/`entryPerimeter=0` pin the connection point without draw.io re-snapping.
- Use `labelBackgroundColor=#FFFFFF` so labels stay readable where they cross lines.

## 5. Verification script (run before previewing)

```python
import xml.etree.ElementTree as ET
tree = ET.parse('component-diagram.drawio'); cells={}
for c in tree.getroot().iter('mxCell'):
    if c.get('edge')=='1': continue
    g=c.find('mxGeometry')
    if g is None: continue
    x,y,w,h=g.get('x'),g.get('y'),g.get('width'),g.get('height')
    if not(x and y and w and h): continue
    cells[c.get('id','')]=(float(x),float(y),float(w),float(h))
def rect(c): x,y,w,h=cells[c]; return (x,y,x+w,y+h)
def ov(a,b): return not(a[2]<=b[0] or b[2]<=a[0] or a[3]<=b[1] or b[3]<=a[1])
def inside(p,c):
    P,C=rect(p),rect(c); return P[0]<=C[0] and P[1]<=C[1] and C[2]<=P[2] and C[3]<=P[3]
```

Confirm: every child fits inside its lane, no unrelated cards overlap, and compute the right `entryY`.

## 6. Iteration discipline

- **When 2–3 rounds of nudging don't fix crowding, RESTRUCTURE, don't patch.** The win that fixed
  the chat lane: `agent_unit` spans the lane top (agent_inner left, tools right), `chat_service`
  compact mid-left, `chat_session` spans the bottom → clean horizontal `tool → search_service`,
  vertical `result_set → agent_inner` up the left corridor, short `tool → twin_data` verticals.
- One conceptual change per round; verify containment before showing.
- **Preserve the user's manual tweaks.** If they edit in draw.io (you'll see attribute reordering,
  `host="Electron"`, custom `exitPerimeter=0`), Read the file back before rewriting.
- TaskCreate/TaskUpdate per round so progress is visible.

## TL;DR

Edit the XML by hand, absolute coords, LIGHT theme with dark title. Reserve arrow corridors
before placing boxes; align connected components on the same y. Render to PNG and actually look
(plus the overlap script) before previewing. Restructure rather than nudge. Big page, big fonts,
big gaps.
